# README模板标准

## 结构化功能描述（第一行要求）
**模块功能的一句话结构化描述** - 必须在第一行用一句话清晰描述该模块的核心功能，格式为：功能描述 - 主要职责 - 适用场景

## 模块概述
简要介绍模块的作用和在整体项目中的位置

## 文件结构
列出模块中的每个文件及其功能说明：

### 核心文件
- `filename.ext` - 文件功能描述

### 配置文件
- `config.json` - 配置说明

### 工具文件
- `utils.py` - 工具函数说明

## 重要定义（Ground Truth）
### 核心功能定义
**重要**: 在此部分定义模块的核心功能和边界，作为所有分析的基准

### 接口定义
**重要**: 定义模块提供的接口和契约

### 数据结构定义
**重要**: 定义关键数据结构和格式

## 代码变更历史
### [日期] - 变更描述
- 变更内容
- 影响文件
- 变更原因

## 依赖关系
- 内部依赖: 列出依赖的其他模块
- 外部依赖: 列出外部库和服务

## 使用示例
提供基本的使用示例和最佳实践
